# 🎓 Student Depression Prediction — ML Classification Project

A supervised machine learning project that predicts depression in students based on academic, lifestyle, and socioeconomic factors. The pipeline covers end-to-end steps: exploratory data analysis, data cleaning, feature engineering, model training, and performance comparison across four classifiers.

---

## 📁 Project Structure

```
student-depression-ml/
│
├── student_depression_dataset.csv          # Raw dataset (27,901 students, 18 features)
├── data-preprocessing-ml-project.ipynb    # EDA, cleaning, feature engineering
├── training-models.ipynb                   # Model training, evaluation & comparison
└── cleaned_student_mental_health.parquet   # Cleaned, model-ready dataset (generated)
```

---

## 📊 Dataset

| Property | Value |
|---|---|
| Source | `student_depression_dataset.csv` |
| Records | 27,901 students |
| Features | 18 (17 input + 1 target) |
| Target | `Depression` (0 = No, 1 = Yes) |
| Class Distribution | ~58.5% depressed / ~41.5% not depressed |

### Features

| Feature | Type | Description |
|---|---|---|
| `Gender` | Categorical | Male / Female |
| `Age` | Continuous | Student's age |
| `City` | Categorical | City of study |
| `Academic Pressure` | Ordinal | Perceived academic workload (scale) |
| `CGPA` | Continuous | Cumulative GPA |
| `Study Satisfaction` | Ordinal | Satisfaction with studies |
| `Sleep Duration` | Ordinal | Hours of sleep per night |
| `Dietary Habits` | Ordinal | Unhealthy / Moderate / Healthy |
| `Degree` | Ordinal | Education level (School → PhD) |
| `Have you ever had suicidal thoughts ?` | Binary | Yes / No |
| `Work/Study Hours` | Continuous | Daily hours spent on work or study |
| `Financial Stress` | Ordinal | Financial pressure level (scale) |
| `Family History of Mental Illness` | Binary | Yes / No |
| `Depression` | **Target** | 0 = No depression, 1 = Depressed |

> Features with low variability (`Profession`, `Work Pressure`, `Job Satisfaction`) and the ID column were dropped during preprocessing.

## 🔧 Notebooks

### 1. `data-preprocessing-ml-project.ipynb` — EDA & Preprocessing

**Exploratory Data Analysis:**
- Dataset shape, column types, and summary statistics
- Missing value detection and duplicate checks
- Distribution analysis (histograms for numeric, bar charts for categorical)
- Outlier detection using the IQR method
- Target variable balance inspection

**Data Cleaning:**
- Replaced ambiguous labels (`"Others"` in `Sleep Duration`, `Dietary Habits`, `Degree`) with `NaN`
- Replaced invalid `"?"` entries in `Financial Stress` with `NaN`
- Removed 22 rows with invalid/noisy `City` values (names, degrees, numbers)

**Feature Engineering:**
- Dropped ID column and three low-information features
- Binary encoding: `Yes → 1`, `No → 0`
- Ordinal encoding for `Sleep Duration`, `Dietary Habits`, `Degree`
- Mode imputation for all remaining missing values
- One-Hot Encoding for `Gender` and `City`
- Dtype optimization (`float32` for continuous, `int8` for binary/ordinal/OHE)
- Saved the clean dataset as `cleaned_student_mental_health.parquet`

---

### 2. `training-models.ipynb` — Model Training & Evaluation

**Setup:**
- Loaded cleaned `.parquet` dataset
- Train/test split: **80% / 20%**, stratified on target
- `StandardScaler` fitted **only on train** data to prevent data leakage (applied to `Age`, `CGPA`, `Work/Study Hours`)

**Models Trained:**

| Model | Tuning Strategy |
|---|---|
| Logistic Regression | Baseline (liblinear solver) |
| Decision Tree | Cross-validated `max_depth` search |
| Random Forest | Cross-validated `n_estimators` × `max_depth` grid |
| SVM | Cross-validated kernel selection + `C` regularization |

**Evaluation Metrics (per model):**
- Accuracy, Precision, Recall, F1-Score, ROC-AUC
- Confusion Matrix with TP / FP / TN / FN breakdown
- Classification Report
- ROC Curves (individual and combined comparison chart)

**Cross-Validation:**
- 5-Fold Stratified K-Fold on training data
- ROC-AUC used as the primary scoring metric for hyperparameter selection
- Final cross-validation comparison across all four models

## 🧪 Models & Results

Four classifiers were trained and evaluated:

- **Logistic Regression** — linear baseline model
- **Decision Tree** — non-linear, interpretable; depth tuned via cross-validation
- **Random Forest** — ensemble of decision trees; `n_estimators` and `max_depth` tuned
- **SVM** — kernel (linear, RBF, poly) and regularization `C` tuned via cross-validation

Performance is compared across Accuracy and ROC-AUC with a combined ROC curve plot and a final test-set summary table.

---

## 🧠 Key Design Decisions

- **Stratified splits** ensure the same class ratio in train and test sets.
- **Scaler fitted only on training data** prevents information leakage from the test set.
- **Mode imputation** used for ordinal/categorical missing values — safe and non-parametric.
- **Parquet format** for the cleaned dataset preserves dtypes and is more efficient than CSV.
- **Cross-validation** used for all hyperparameter tuning — no separate validation set needed.
